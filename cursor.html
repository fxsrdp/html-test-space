Custom Proximity Cursor

Ein Cursor, der auf interaktive Elemente reagiert. Hover-Glow über Links, N-Body-Physik-Split über Drag-Zonen. Reines CSS + Vanilla JS, kein Framework.

← zieh die Maus hier rüber → Cursor split in 3 Punkte

hover über Links — proximity glow

entfern dich langsam — fade-out mit Distanz

Drei Verhaltensmodi

○

Idle

Transparenter Ring folgt der Maus. Opacity skaliert mit Nähe zu Links (0.15–0.4).

●

Hover

Solider Cyan-Kreis wächst auf 1.4×. Leuchtet bei Link-Nähe <60px.

∴

Split

3 Punkte springen auseinander. N-Body-Gravitation hält sie zusammen. Kollisions-Abstoßung verhindert Überlappung.

How We Got Here

Architecture

css/cursor.css — 15 Zeilen. cursor:none auf allen Elementen. Ein #cur-Div mit 3 .dot-Kindern. Klassen .on, .split, .vis, .rejoin steuern Zustände.

js/cursor.js — 149 Zeilen. Event-Listener für Mausbewegung. Proximity-Detection via edgeDist() (kürzeste Distanz zu Element-Rect). Physik-Loop via requestAnimationFrame.

N-Body Physics (Split-Modus)

3 Punktmassen mit Gravitation zueinander und zum Zentroid. Dämpfung (damp=0.997), Abstoßung (repel=800) bei Distanz <10px, Cursor-Trägheit (cursorVX*0.005), Zentripetalkraft zum Ursprung. Timestep dt=0.8 für Stabilität.

Prototype Evolution

v1 — Orange (#fe5000) · felixschreiber-site cursor.css + cursor.js
v2 — Orange (#fe5000) · designpractice site (gleicher Code)
v3 — Cyan (#00e5ff) · html-test-space, angepasst für dark theme

Key Decisions

— CSS-only idle/hover states: keine JS-Animation für Glow
— requestAnimationFrame statt setInterval: 60fps-synchron
— edgeDist() statt contains(): erkennt Nähe, nicht nur Hover
— Cooldown nach Split (3s): verhindert Dauer-Oszillation
— Hardware-Cursor versteckt (cursor:none): kompletter custom stack

Browser Support

Alle modernen Browser ✅ — cursor:none, requestAnimationFrame, CSS transitions sind universell. Keine Polyfills nötig.

Performance

<1 KB gzipped JS. 3 DOM-Reads pro Frame (getBoundingClientRect für Drag-Ziele). Physik-Loop nur aktiv während Split (sonst physRunning=false). Idle-Modus: 1 transform-Update pro Mausbewegung.
